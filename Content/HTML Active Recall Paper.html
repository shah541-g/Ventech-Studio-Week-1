Answer Key Kholein (sirf self-check ke liye — pehle khud attempt karein)
Part 1
1. HyperText Markup Language. HTML sirf structure/layout nahi, poori website ka look/feel bhi decide karta hai.
2. False — word processors HTML ke liye overkill hain. Windows→Notepad, Mac→TextEdit, jo aapko fundamentals par focus karne dete hain kyunke har tag khud type karna padta hai.
3. <!DOCTYPE html><html><head><title>My Site</title></head><body>Hello</body></html>. index.html special hai kyunke browser ise root folder mein home page samajhta hai. <p>, body ke andar hone ki wajah se html ka grandchild hoga.
4. B) <sub>. HTML whitespace ignore karta hai — layout tags ke order se decide hoti hai. h1 sab se bara, h6 sab se chota.
5. <!-- TODO: fix navbar -->. Use cases: notes/reminders chorna, aur code temporarily disable (comment-out) karna.
6. Paragraph ka apna red background dikhega kyunke child styling parent styling ko override karti hai. Poore system ka naam CSS (Cascading Style Sheets).
7. Section, article/page ko chote logical parts mein baantne ke liye; aside content jo main content se direct related nahi (jaise ad). Semantic tags readability/organization aur SEO dono ke liye behtar hain.
8. <a href="https://google.com" target="_blank">Google</a>. Absolute URL doosri websites ke liye (poora address), relative URL apni hi site ke andar doosre pages ke liye.
9. Alt text image load na hone par backup text dikhata hai — accessibility ke liye zaroori. Resize karte waqt original aspect ratio (3:2) maintain karna zaroori hai warna image stretch/squeeze hogi.
10. poster attribute. Local video <video> tag se file directly embed karti hai; YouTube embed <iframe> se YouTube ke apne player ke sath aata hai.
11. C) i, ii, iii. <dt> = description term (item), <dd> = uski explanation/detail.
12. colspan="2" ek td ko 2 columns ki jagah lene deta hai, baaki entries shift ho jati hain. th = header (bold), td = actual data.
13. Block = poori width leta hai, agla element neeche jata hai (div); Inline = sirf zaroori space leta hai, sath rehta hai (span). Div overall containers ke liye, span chote inline text pieces ke liye.
14. JavaScript chahiye hoti hai inputs ko functional banane ke liye — HTML sirf UI define karta hai. Checkbox: multiple options select ho sakte hain; Radio (same name): sirf ek option select ho sakta hai.
15. Security ke liye — koi malicious website unki site ko iframe mein daal kar phishing-type attack kar sakti hai.
16. Viewport meta tag mobile devices par website sahi display hone ke liye zaroori hai, warna site zoomed-out/desktop version mein dikhegi. Ideal length ~160 characters.
Part 2
17. HTML = kya hai (structure), CSS = kaisa dikhta hai (presentation), JS = kya karta hai (behavior). HTML→DOM, CSS→CSSOM.
18. Label ka for id se match hona chahiye — is se label pe click karne se input focus/select ho jata hai (accessibility ke liye important). B) button type="button".
19. <input type="email" required>. Client-side validation JavaScript se bypass ki ja sakti hai, is liye server-side validation security ke liye zaroori hai.
20. Screen readers layout-only tables ko "data table" samajh kar confusing tareeqe se parhte hain, jo actual structure ko misrepresent karta hai aur SEO ke liye bhi bura hai.
21. Landmarks semantic tags (header/nav/main/aside/footer) hain jo screen reader users ko shortcut key se seedha "next landmark" (jaise main content) pe jump karne dete hain. Checklist: alt text, label, hierarchy mein.
22. DNS domain name (jo insaan yaad rakhte hain) ko IP address (number) mein convert karta hai, bilkul phone book ki tarah naam se number dhoondna. Request parts: URL, method, headers, body.
23. Normal pattern mein client hamesha pehle request bhejta hai; WebSockets mein server khud persistent connection par client ko naya data push kar sakta hai (real-time). Mobile apps JSON mangte hain.
24. Switch same LAN ke andar devices ko connect karta hai; Router alag-alag networks ko aapas mein aur internet se connect karta hai. Wireless switch = Access Point.
25. Single point of failure avoid karna, cable length impractical hona, aur load balancing — isliye distributed routers. Continents ke darmiyan undersea fiber-optic cables se traffic jata hai.
26. Tunneling: original packet encrypt karke doosre packet ke andar "pack" kar diya jata hai (letter ko envelope mein daalne jaisa) taake public internet par safe rahe. Private WAN zyada secure lekin costly; Public WAN (VPN) kam secure nahi (encryption ki wajah se acha) aur sasta/common.
27. Local ISP = mohalla level, Regional = country ke cities connect karta hai, Global = countries/continents (undersea cables). Peering = bari companies ka Local ISP ke sath direct connection, taake poori hierarchy se guzarne ki bajaye traffic seedha pohanche.
28. Raw Bytes → Character Encoding → Tokenization → Object Creation → DOM Construction. JS DOM/CSS ko modify kar sakti hai isliye browser pehle JS process karta hai jo rendering rok deta hai — solution: async/defer attributes se JS ko end mein/asynchronously load karna.
Part 3
29. TCP reliable hai (order + delivery guarantee, resend on loss), UDP fast hai lekin reliability guarantee nahi karta. Socket = IP address + Port number. Port 80 = HTTP, Port 443 = HTTPS.
30. Certificate = server ki identity prove karta hai (CA se signed); Handshake = encryption algorithm aur connection security decide karna; Encryption = actual data ka secure transmission. CA ek trusted third-party hai jo certificate sign karti hai.
31. Persistent connections har request ke liye naya TCP connection kholne ka overhead khatam karti hain. HTTP/0.9 mein sirf GET tha, koi headers/status codes nahi thay. Head-of-line blocking: ek connection par ek waqt sirf ek request outstanding ho sakti thi, slow request baaki ko rok deti thi.
32. SPDY ka idea: bandwidth badhane ka fayda plateau ho jata hai, lekin latency kam karna hamesha performance behtar karta hai. Multiplexing head-of-line blocking khatam karti hai — ek hi connection par multiple requests parallel chal sakti hain. True.
33. Recursor = librarian jo poori book dhoondta hai; Root = index batata hai kaunse rack (TLD) par jana hai; TLD = specific rack (.com); Authoritative = final dictionary jo asal IP deta hai. 8 steps: browser→resolver→root→TLD→authoritative→IP wapis→browser ko IP milta hai→HTTP request bhejta hai.
34. Recursive query: server se poora record dhoond kar dene ki maang; Iterative: server best jawab ya agle server ka reference deta hai, client khud agla step karta hai. Caching 3 levels: Browser cache, OS-level cache, ISP resolver cache. TTL = Time To Live.
